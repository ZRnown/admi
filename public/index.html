使用说明

  • 一个转发实例对应一个 Discord Token，可选择机器人 Token 或自用号 Token。
  • 每个转发实例都可以单独配置转发规则、关键词、排除/替换规则。
  • 转发时如需伪装为源用户，请勾选「使用源用户昵称和头像」（Discord→Telegram 仅使用昵称）。
  • 排除关键词会直接删除命中的词，而不会阻止整条消息转发。
  • 可以添加多个转发实例并行运行，后端会为每个实例独立登录并转发。
  • 转发类型说明：
  • • Discord → Discord：填 Discord Token，规则中填写来源频道 ID 与目标 Webhook URL。
  • • Discord → Telegram：填 Discord Token + Telegram Bot Token，Bot 需在目标群/频道具备发消息权限，规则中填写 Discord 频道 ID 与 Telegram Chat ID。
  • • Telegram → Discord：填 Telegram API ID/Hash 并完成 Session 认证，规则中填写来源 Chat ID 或用户名（@xxx/xxx）与 Discord Webhook URL。
  • • Telegram → Telegram：填 Telegram Bot Token 与 API ID/Hash（用于接收与转发），规则中填写来源 Chat ID 与目标 Chat ID，可在规则内选择“机器人/用户”发送。
  • • Discord → 飞书：填 Discord Token，配置飞书 Webhook 或线程 ID，若需图片/视频转发请填写飞书 App ID/Secret 并开通相关权限。
  • • 限制可选类型：在 `.env` 设置 `ENABLED_FORWARDING_TYPES=discord-to-discord,discord-to-telegram` 等，留空表示全部可用。
  • 关于 Token：
  • • 此 Token 与 Discord Bot Token 不同。
  • • 此 Token 在您更改密码时会发生变化。
  • • 如果启用了双因素认证（2FA），需要先禁用它才能获取 Token。
